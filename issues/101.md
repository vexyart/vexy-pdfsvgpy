There must be a way to specify the FORMATS with --vinput, --voutput, --binput, --boutput. 

For input like --binput png_d vs. --binput png_s — that determines what the the input and output paths represent. For ..._d formats, the input path should be a folder (non-recursive search), or if --recursive is given, a recursive search, or --glob is given, a glob pattern. If --binput is jpg_d, we should look for .jpg .jpeg .JPG .JPEG. We should also support different casings for the other formats. 

For output like --voutput pdf_m vs. --voutput pdf_l — that determines the type of processing that needs to happen. If -m, we need to combine all documents into a single multi-page PDF. If -l, we need to overlay all documents to one single-page PDF documents with layers.
